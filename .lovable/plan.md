## الخطة: سجلّ المحتوى الموحّد في لوحة المالك

إضافة تبويب جديد في صفحة `/owner/content` (أو صفحة منفصلة `/owner/registry`) يعرض كل المحتوى المُنشأ (فيديوهات + مقالات + منشورات) في **جداول منظّمة** تبدأ بـ `short_id` كمعرّف رئيسي للسجلّ.

### الأعمدة الموحّدة في كل جدول

| العمود | الوصف |
|---|---|
| `short_id` | المعرّف القصير (a123456) — قابل للنسخ + رابط مباشر |
| التاريخ | `created_at` بصيغة مقروءة |
| النوع/المصدر | يوتيوب / مقال داخلي / منشور / إلخ |
| الصورة | thumbnail / featured_image / أول media_url |
| العنوان | مع رابط للصفحة الفعلية |
| المالك | avatar + username من profiles |
| الإحصائيات | views / likes / comments |
| الحالة | منشور / مسودّة / مخفي |
| الإجراءات | معاينة • نسخ الرابط • حذف |

### تبويبات الجدول

1. **الفيديوهات** — من `channel_videos` (short_id, video_id, platform, thumbnail, channel_id…)
2. **المقالات** — من `articles` (short_id, slug, featured_image, status, category…)
3. **المنشورات** — من `posts` (short_id, type, media_urls، أول صورة كـ thumbnail)
4. **الكل** — دمج الثلاثة مع عمود إضافي "النوع"

### الميزات

- بحث بالـ `short_id` أو العنوان أو اسم المالك
- فلترة حسب التاريخ والحالة والمصدر
- ترقيم صفحات (20 لكل صفحة)
- زر نسخ سريع للـ `short_id` والرابط الكامل
- زر معاينة يفتح الصفحة العمومية في تبويب جديد
- زر حذف (يكتب في `owner_audit_logs`)
- إحصائيات أعلى الصفحة: إجمالي كل نوع

### التنفيذ التقني

- **ملف جديد**: `src/routes/_owner/owner.registry.tsx` (أو إضافة تبويبات لـ `owner.content.tsx`)
- استخدام `OwnerShell` و`OwnerCard` الموجودين للحفاظ على نفس التصميم
- استعلامات `useQuery` منفصلة لكل جدول مع `count: "exact"` للترقيم
- جلب profiles بالدفعات (batch) لتفادي N+1
- استخدام مكوّن `Table` من shadcn للجداول
- إضافة رابط جانبي جديد في `OwnerShell` بعنوان "سجلّ المحتوى" أو دمجه في "المحتوى" الموجود

### القرار المطلوب

- هل أُضيفه كتبويبات داخل `owner.content.tsx` الحالي (أنظف)، أم كصفحة مستقلة `/owner/registry`؟
- إذا فضّلت صفحة مستقلة، سأحتفظ بصفحة المحتوى الحالية كما هي.
